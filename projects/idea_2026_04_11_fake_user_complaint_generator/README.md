# 假装用户吐槽生成器

一个轻量的静态网页 playground：输入产品、功能或页面描述，快速生成 5 条“像真实用户会说出来”的吐槽。

## 它能做什么

这个工具会根据输入内容，模拟用户的第一反应，输出：

- 5 条不同风格的吐槽
- 每条吐槽背后的原因
- 这一轮暴露出的常见问题
- 下一步最该补的动作建议

它不是为了恶意攻击产品，而是为了提前看见：
- 用户会困惑什么
- 用户最容易质疑什么
- 哪些文案一看就像“空话”
- 哪些证据或示例缺失会让人不信

## 适合场景

- landing page 初稿自检
- MVP 想法过筛
- 新功能描述打磨
- 上线前先做一轮“反向审稿”

## 当前版本特点

这是一个 **纯前端、固定规则、可解释** 的版本：

- 不联网
- 不调用外部模型
- 不读取真实用户评论
- 只根据输入中的语义线索，模拟常见用户抱怨方式

## 使用方式

直接用浏览器打开 `index.html`。

你可以：
- 输入产品或功能描述
- 可选填写目标用户
- 选择吐槽强度
- 点击“生成吐槽”
