# 功能优先级吵架模拟器

一个轻量的静态网页 playground：输入 3 个候选功能，模拟产品、用户、工程三方对优先级的争论，然后给出一个更务实的排序建议。

## 它能做什么

输入 3 个功能后，这个工具会输出：

- 产品视角怎么看
- 用户视角最在乎什么
- 工程视角最怕什么
- 一段“模拟吵架现场”
- 最终优先级建议（P1 / P2 / P3）

## 适合场景

- MVP 阶段做功能取舍
- roadmap 初稿排序
- 团队内部对优先级争不清时先做一次外显化讨论
- 帮自己避免“什么都想做”的扩张冲动

## 当前版本特点

这是一个 **纯前端、固定启发式规则** 的版本：

- 不联网
- 不调用外部模型
- 不读取真实项目数据
- 主要根据关键词和上下文线索做一个“够用的优先级争论模拟”

所以它更像一个决策辅助 playground，而不是严肃的项目管理系统。

## 使用方式

直接用浏览器打开 `index.html`。

你可以：
- 输入 3 个候选功能
- 可选补充当前阶段背景
- 点击“开始吵架”

## 后续可演进方向

- 支持更多角色（老板 / 销售 / 设计）
- 支持给每个功能手动设置实现成本
- 支持导出为优先级说明文档
- 支持根据产品阶段切换不同权重
